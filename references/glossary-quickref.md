# 跨书核心术语速查

## 体系结构与性能

| 术语 | 书中定义 | 常见误解 |
|---|---|---|
| **Quantitative Approach** | 用公式、基准测试、成本模型评估设计 | 停留在"更快/更慢"的感性判断 |
| **Amdahl's Law** | 加速比受限于可加速部分占比 | 以为加核就能无限加速 |
| **Locality** | 时间局部性 + 空间局部性 | 地理或概念上的"邻近" |
| **Memory Hierarchy** | 寄存器→缓存→主存→磁盘的金字塔结构 | 视为纯硬件细节 |
| **ILP / TLP / DLP** | 指令级/线程级/数据级并行 | 并行 = 多线程 |

## 网络

| 术语 | 书中定义 | 常见误解 |
|---|---|---|
| **End-to-End Principle** | 端系统才能准确判断语义正确性 | 所有功能都应下沉到网络层 |
| **AIMD** | 加性乘性减小，公平探测可用带宽 | 拥塞控制 = 静态限速 |
| **ARQ** | 自动重传请求（ACK + 超时 + 序列号） | 可靠传输由底层网络保证 |

## 并发与一致性

| 术语 | 书中定义 | 常见误解 |
|---|---|---|
| **2PL / SSI / MVCC** | 并发控制的不同策略范式 | 锁越多越好 |
| **Linearizability** | 每个操作看起来在调用瞬间原子完成 | 与 Serializability 混淆 |
| **CAP** | 分区时一致性 vs 可用性的不可兼得 | 系统固有属性而非设计选择 |
| **Vector Clock** | 可检测并发冲突的偏序关系 | 与 Lamport Timestamp 混淆 |

## 数据与存储

| 术语 | 书中定义 | 常见误解 |
|---|---|---|
| **WAL** | 先写意图日志，再异步刷回 | 日志只是审计用途 |
| **LFS** | 追加日志结构，顺序写优先 | 日志 = 慢 |
| **COW** | 写时复制，不覆盖旧块 | 浪费空间 |
| **RSM** | Reliability / Scalability / Maintainability | 只关注可靠性 |

## 抽象与验证

| 术语 | 书中定义 | 常见误解 |
|---|---|---|
| **Invariant** | 状态机在所有可达状态下保持为真的谓词 | 不变量 = 单元测试断言 |
| **Safety** | "坏事情永远不会发生" | 系统总是安全的 |
| **Liveness** | "好事情最终会发生" | 与可用性完全等同 |
| **Metalinguistic Abstraction** | 用元语言（解释器/DSL）控制复杂度 | DSL 只是语法糖 |
